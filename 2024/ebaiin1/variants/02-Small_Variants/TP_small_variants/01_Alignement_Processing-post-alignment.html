I - Alignement

BWA est un logiciel d'alignement dont la particularit est d'utiliser la transforme de Burrows-Wheeler (Burrows-Wheeler Aligner) ce qui lui permet d'utiliser peu de mmoire.

On commence par charger le module bwa. Comment trouver le module bwa ? (allez sur le terminal)

In[]:
# Cette commande ne fonctionne pas ici il faut aller dans un terminal pour la lancer
#module avail -l bwa
In[]:
module load bwa/0.7.17
In[]:
# Afficher l'aide de l'algorithme mem et se rendre dans le dossier tp_variant
bwa | cat

1. Indexation du gnome

BWA a besoin de genome de ref index. pour cela on va utiliser la fonction index de bwa.

In[]:
# Crer l'index du gnome pour bwa
# bwa index # pour afficher l'aide
# usage: bwa index <fasta_file>
# remplacer <fasta_file> par le fasta du gnome de rfrence.
bwa index genome/Bos_taurus.UMD3.1.dna.toplevel.6.fa

2. Alignement des reads sur le gnome de rfrence

BWA propose plusieurs algorithme d'alignement ... Celui qui est recommand pour les donnes short-reads est mem (Maximum exact match)

Afficher d'abord l'aide de bwa mem

In[]:
bwa mem | head
In[]:
# usage: bwa mem -t 4 -R <readGroup> <fasta_file> <fastq1> <fastq2> > <sam_output>
# -R "@RG\tID:1\tPL:Illumina\tPU:PU\tLB:LB\tSM:SRR1262731"
bwa mem -t 4  \
    genome/Bos_taurus.UMD3.1.dna.toplevel.6.fa  \
    fastq/SRR1262731_extract_R1.fq.gz  \
    fastq/SRR1262731_extract_R2.fq.gz  >  SRR1262731_extract.sam

Le caractre "\" permet de revenir la ligne sans interrompre la commande. Il faut veiller ne rien crire aprs, y compris des espaces.

Quelle est la taille du fichier SRR1262731_extract.sam

In[]:
ls -lh SRR1262731_extract.sam

2.1 Alignement long-reads

Pour les long-read l'outil d'alignement le plus courament utilis est minimap2. Il utilise un sytme d'index et un algorithme mieux adapt pour grer les homopolymres les taux d'erreurs plus levs lis aux long-read.

L'outil propose des "preset" de paramtres en fonction de la technologie (PacBio, Nanopore ...) : https://lh3.github.io/minimap2/minimap2.html#8

# exemple
minimap2 -ax map-ont ref.fasta reads.fastq.gz > aln.sam

b) Ajout de la provenance des chantillons (readGroup)

  • ReadGroups (RG) : permet d'associer des informations sur la provenance des reads
    • Identit : numro de run ou d'chantillon
    • Squenage, type de librairie, ...

Ces informations sont ncessaires la recherche de variants

Plus d'informations sur les ReadGroups

Comment vrifier la prsence de ReadGroups dans un fichier BAM ?

In[]:
# Afficher l'entte du bam (-H)
samtools view -H SRR1262731_extract.bam

Comment pouvons-nous l'ajouter ?

In[]:
# usage : samtools addreplacerg -r <readGroup> -o <output_bam> <input_bam>
samtools addreplacerg -@ 4 -r "@RG\tID:1\tPL:Illumina\tPU:PU\tLB:LB\tSM:SRR1262731" \
    -o SRR1262731_extract.rg.bam \
    SRR1262731_extract.bam

Vrifions en filtrant avec grep

In[]:
samtools view -H SRR1262731_extract.rg.bam | grep '^@RG'

Il est ncessaire de prparer les donnes avant de pouvoir dtecter les variants

Rappel du workflow :

1. Marquage des duplicats de PCR

Identifier les reads provenant d'une mme molcule issus de :

  • PCR duplicates : amplification PCR durant la prparation de la librairie
  • Optical duplicates : Cluster Illumina identifi comme deux clusters

  • Garder les duplicats : probabilit importante de confondre les duplicats avec des fragments biologiques issus du mme locus
  • Marquer les duplicats mais les conserver dans le fichier BAM
  • Supprimer les duplicats du fichier BAM : Certains outils les supprimeront par dfaut (samtools, GATK, ...)

Avec l'outil MarkDuplicates de la suite PicardTools intgre la suite GATK4

In[]:
# Charger le module gatk4 en 4.2.3.0
module load gatk4/4.2.3.0
In[]:
# Afficher l'aide de gatk MarkDuplicates
gatk MarkDuplicates --help | head
In[]:
# Lancement de l'outils
gatk MarkDuplicates --java-options '-Xmx8G' \
    --INPUT SRR1262731_extract.sort.bam \
    --OUTPUT SRR1262731_extract.sort.md.bam \
    --METRICS_FILE SRR1262731_extract_metrics_md.txt \
    --VALIDATION_STRINGENCY SILENT  2>log/MarkDuplicates.log
In[]:
# Afficher les 20 premires lignes du fichier
head SRR1262731_extract_metrics_md.txt
In[]:
# Rechercher le % de duplicat de PCR
grep -A1 "LIBRARY" SRR1262731_extract_metrics_md.txt

2. Filtres sur les alignements

Restreindre le fichier BAM en fonction de mtriques d'alignements :

  • Qualit de mapping (MAPQ) suffisante
  • Retrait des reads non mapps
In[]:
# Suppression des reads non mapps et filtre sur les reads avec MAPQ < 30
samtools view -bh -F 4 -q 30 SRR1262731_extract.sort.md.bam \
    > SRR1262731_extract.sort.md.filt.bam

Pour utiliser le paramtre -F : plus d'information sur les SAM flags

In[]:
# Rsum de l'ensemble des flag prsents dans notre BAM
samtools flagstat SRR1262731_extract.sort.md.filt.bam \
    > SRR1262731_extract.filt.flagstat.txt
In[]:
# Visualiser le fichier
cat SRR1262731_extract.filt.flagstat.txt

Restreindre le fichier BAM en fonction de mtriques d'alignements (suite) :

  • Alignements intersectant les rgions d'intrt
  • En fonction du nombre de mismatchs, de la taille d'insert, de paires mappes sur des chromosomes diffrents, ...
In[]:
# Conservation des alignements dans les rgions cibles
module load bedtools/2.29.2
In[]:

In[]:
